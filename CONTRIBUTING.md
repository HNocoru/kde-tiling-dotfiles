## Development Setup

1. Fork the repository
2. Clone your fork:
   ```bash
   git clone https://github.com/YOUR-USERNAME/kde-tiling-dotfiles.git
   cd kde-tiling-dotfiles
   ```
3. Create a branch:
   ```bash
   git checkout -b feature/your-feature-name
   ```
4. Make your changes
5. Test thoroughly
6. Commit with descriptive messages
7. Push and create a pull request

## Coding Standards

### Configuration Files
- Add comments explaining each section
- Follow existing formatting
- Document default values
- Explain trade-offs where applicable

### Scripts
- Include error handling
- Add helpful output messages
- Test on clean system
- Support dry-run mode where applicable

### Documentation
- Use clear, concise language
- Include examples
- Add screenshots when helpful
- Keep formatting consistent

## Pull Request Process

1. **Test your changes** on a clean KDE install if possible
2. **Update documentation** if you changed behavior
3. **Describe your changes** clearly in the PR description
4. **Reference related issues** if applicable
5. **Be responsive** to feedback and review comments

### PR Checklist
- [ ] Changes tested locally
- [ ] Documentation updated
- [ ] Comments added to code
- [ ] CHANGELOG.md updated (if applicable)
- [ ] No merge conflicts
- [ ] Follows project conventions
